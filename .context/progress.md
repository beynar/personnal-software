## Codebase Patterns

- **TanStack Start v1.167+ uses Vite directly** ÔÇö no vinxi, no app.config.ts. Config goes in `vite.config.ts` with `tanstackStart()` plugin from `@tanstack/react-start/plugin/vite`.
- **srcDirectory defaults to `"src"`** ÔÇö must set `srcDirectory: "app"` in tanstackStart() options to use `app/` directory.
- **Router entry must export `getRouter()`** ÔÇö not `createRouter`. The plugin resolves `router.tsx` in srcDirectory and expects this named export.
- **Entry files resolved by plugin**: `router.tsx` (required), `client.tsx` (optional), `server.ts` (optional), `start.tsx` (optional). Extensions tried: `.ts`, `.js`, `.mts`, `.mjs`, `.tsx`, `.jsx`.
- **Cloudflare vite-plugin**: Do NOT set `main` in `wrangler.toml` ÔÇö the plugin handles it. Setting `main` causes "doesn't point to an existing file" error.
- **Biome v1.9.4**: Uses `files.ignore` array to exclude files. JSON formatter uses separate `json.formatter` config with `indentStyle: "space"`. Schema is `1.9.4`.
- **routeTree.gen.ts**: Auto-generated by TanStack router plugin. Exclude from biome and gitignore.
- **Tailwind v4**: Uses `@import "tailwindcss"` in CSS, no `tailwind.config.js` needed. Import CSS via `?url` suffix in root route head links.
- **Path alias `~` ÔåÆ `app/`**: Defined in both `tsconfig.json` (`paths`) and `vite.config.ts` (`resolve.alias`). Both must be kept in sync. Vite uses `fileURLToPath(new URL("./app", import.meta.url))`.
- **shadcn/ui setup (manual)**: Components in `app/components/ui/`, cn helper in `app/lib/utils.ts`, config in `components.json`. Radix UI primitives installed per-component. Tailwind v4 theming uses `@theme inline` block + CSS variables in `:root`.
